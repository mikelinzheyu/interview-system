# 📱 如何访问 /home 页面

## 问题

访问 `http://localhost:5174/home` 时出现"找不到网页"错误。

## 原因

`/home` 路由需要用户认证 (`requiresAuth: true`)。当用户未登录时，应用会自动将请求重定向到 `/login` 页面。

## 解决方案

### 步骤 1：访问登录页面

访问以下任一 URL：
- `http://localhost:5174/login` (直接登录页)
- `http://localhost:5174/` (会自动显示登陆/注册选项)

### 步骤 2：使用测试凭证登录

Mock 服务器接受任何用户名和密码组合。可以使用：

**推荐用户名**：`testuser`
**任何密码**：例如 `password` 或 `123456`

或者使用其他用户名/密码，都会成功登录。

### 步骤 3：登录成功后访问 /home

登录成功后，应用会自动跳转到 `http://localhost:5174/home`。

或者你也可以在登录后手动访问：`http://localhost:5174/home`

## 📝 详细步骤

### 方式 A：通过登录页

```
1. 打开浏览器
2. 访问：http://localhost:5174/login
3. 输入：
   - 用户名：testuser (或任意名称)
   - 密码：password (或任意密码)
4. 点击"登录"按钮
5. ✅ 自动跳转到 /home 页面
```

### 方式 B：直接访问根路由

```
1. 访问：http://localhost:5174/
2. 点击"登录"或"注册"
3. 输入任意用户名和密码
4. 点击"登录"
5. ✅ 自动跳转到 /home 页面
```

## 💡 常见问题

### Q: 为什么总是跳转到 /login？
A: 说明用户未登录。需要先完成登录流程，才能访问受保护的页面。

### Q: 密码随意填写可以吗？
A: 可以！Mock 服务器接受任何用户名/密码组合。

### Q: 登录后能访问其他用户的数据吗？
A: 不能。Mock 服务器统一返回 `testuser` 的数据，无论用户登录时使用什么凭证。

### Q: 如何退出登录？
A: 在应用页面找到用户菜单，点击"退出登录"或"登出"按钮。

### Q: token 会过期吗？
A: 在 Mock 服务中，token 24 小时后过期。实际应用中请根据服务器设置。
