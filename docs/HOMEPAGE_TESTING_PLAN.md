## 🧪 详细测试用例

### 用例 1: HeroSection 组件显示

**前置条件**: 已登录用户

**测试步骤**:
1. 打开首页 `/`
2. 观察 Hero Section 区域

**预期结果**:
- ✓ 显示欢迎文案 "欢迎回来{用户名}！"
- ✓ 显示副标题 "准备好开始您的下一次面试了吗？"
- ✓ 显示 3 个按钮: 开始面试、浏览题库、查看排名
- ✓ 背景为紫蓝渐变
- ✓ 有装饰圆形在背景
- ✓ 整体居中对齐

### 用例 2: StatsCard 卡片显示

**前置条件**: 首页已加载

**测试步骤**:

**预期结果**:
- ✓ 显示 4 个卡片: 面试次数、练习时长、平均分数、当前排名
- ✓ 每个卡片显示图标 (64×64px)
- ✓ 显示数值和单位
- ✓ 显示趋势信息 (↗/↘ + 百分比)
- ✓ 卡片背景为白色透明 (rgba)
- ✓ 卡片有阴影效果

## 🛠️ 测试工具

### 浏览器开发工具
```
Chrome DevTools:
  1. 打开: F12 或 Ctrl+Shift+I
  2. Elements 标签: 检查 HTML 结构
  3. Styles 标签: 检查 CSS 样式
  4. Console 标签: 检查错误信息
  5. Performance 标签: 性能分析
  6. Responsive Design Mode: Ctrl+Shift+M (响应式测试)
```

### 在线工具
- Lighthouse: https://developers.google.com/web/tools/lighthouse

## 🚀 测试执行

### 本地测试运行

```bash
# 1. 进入项目目录
cd frontend

# 2. 安装依赖 (如需要)
npm install

# 3. 启动开发服务器
npm run dev

# 4. 打开浏览器访问
http://localhost:5173

# 5. 打开 DevTools (F12)
# 6. 在 Console 标签中查看是否有错误
# 7. 依次测试各个用例
```

### 响应式测试

```bash
# 在 Chrome DevTools 中:
1. 按 Ctrl+Shift+M 打开响应式设计模式
2. 选择不同的设备尺寸测试
3. 观察布局变化是否合理
```

### 性能测试

```bash
# 在 Chrome DevTools 中:
1. 打开 Performance 标签
2. 点击录制按钮
3. 刷新页面
4. 停止录制
5. 查看 FCP、LCP、长任务等指标
```
